1. 介绍 HP LoadRunner 概念

是什么东西？干什么用的？谁来用？在什么场景下用？

  • 惠普软件业务部开发的软件测试工具。
  • 用于测试应用程序（B/S、C/S 结构都可以），测量负载下的系统行为和性能。LoadRunner可以使用应用程序软件同时模拟数以千计的用户，记录和稍后分析应用程序关键组件的性能。
  • LoadRunner 通过在应用程序组件之间生成消息或模拟与用户界面的交互（如按键或鼠标移动）来模拟用户活动。要生成的消息和交互被存储在脚本中。LoadRunner可以通过记录脚本来生成脚本，例如在客户端Web浏览器和应用程序的Web服务器之间记录HTTP请求。（生成脚本的两种方式，1. 通过录制在窗口上的操作 2. 自己编写脚本）
  • 专业的测试人员
  • 应用程序压力测试时使用

2. 介绍 LoadRunner 的重要的组件

由哪几部分组成，各自完成什么工作？由 5 个组件组成，各自分工明确，可以独立安装和运行。包括：

  • irtual User Generator 录制最终用户业务流程并创建自动化性能测试脚本，即 Vuser 脚本。
  • Load Generator 通过运行 Vuser 产生负载。
  • Controller 组织、驱动、管理并监控负载测试。指定使用哪个脚本，多长时间等等。在运行期间，控制器接收实时监视数据并显示状态
  • Analysis 用于查看、剖析和比较性能结果。产生的测试报告。
  • Launcher 使您可以从单个访问点访问所有 LoadRunner 组件

3. LoadRunner 简单使用

基于 LoadRunner 的测试流程

规划、创建脚本、定义场景、执行场景和分析结果，例如并发用户数量、典型业务流程和要求的响应时间。

  1. 创建 Vuser 脚本。在自动化脚本中录制最终用户活动。
  2. 定义场景。使用 LoadRunner Controller 设置负载测试环境。
  3. 运行场景。使用 LoadRunner Controller 驱动、管理并监控负载测试。
  4. 分析结果。使用 LoadRunner Analysis 创建图和报告并评估性能。

创建脚本在 Virtual User Generator (VuGen) 中
定义场景、执行场景在 LoadRunner Controller 中
分析结果在 Analysis 中

重要的概念

录制
演示一下录制的过程，需要注意的点:

  • Win10 下可能不能录制，无法打开 IE
  • 需要破解、激活才能真正的录制

在正常情况下，录制登陆、新增等业务操作，我们录制的是什么呢？
录制的是Http发送给服务器的请求，HTTP请求包含请求头、主体等信息（当然有的时候，可能请求中没有主体），服务器收到请求后，给予一定的响应。

事务
首先要了解Loadrunner的事务指的是什么？事务就是对请求进行分组，目的是对1个或多个请求汇总其响应时间等指标。
为什么需要对多个请求汇总成事务呢？因为我们往往想考查一个功能（例如查询）的性能情况，但其实查询这个功能需要发送多个HTTP请求，因此需要把多个请求进行汇总。

参数化
演示如何参数化，
脚本中的参数化就是将其中固定的值（或者录制的值）通过参数修改为可变的值，这个过程就是参数化；参数可以由 LR 自动生成、指定文本文件读取、指定数据库。

思考时间
表示您在各步骤之间等待的实际时间，可用于模拟负载下的，快速和慢速用户操作。“思考时间”这种机制可以让负载测试更加准确地反映实际用户操作。
