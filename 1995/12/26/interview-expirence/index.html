个人的面试经历小结

1、全业科技（杭州）

时间：2016年12月26日 现场面试

说实话开始之前一点也不紧张…..先做面试题，好几题没有做出来。

老板不在，一个看起来傻傻的员工（具体也不知道什么置为）面试。说起来是同一个学校的学长，还是实验室老师的学生。顿时心中的戒备心就放下了。

接下来的聊天也是比较顺利，没问道什么高深的技术，大致就讲了项目中用到了哪些框架。大概十来分钟，面试的前辈没有很多的眼神交流。感觉很腼腆，呆呆的。谁会知道我的第一次面试竟会这般。

两天后没有给我答复，遂发短信询问，很遗憾未通过。

总结

自己也不是很想去这个公司，办公氛围，还有公司装修整体给人一种压抑的感觉，是那种传统的外包公司。
那天坐公交还坐吐了，人生第一次。
第一个面试通知，不管怎样还是得感谢公司提供的面试机会。

2、杭州容方信息技术有限公司（杭州）

时间：忘记了 电话面试

打电话过来的时候，在吃饭。也没介绍自己公司，上来就是一顿询问，实习只有2k，问我去不去。我要求把公司的情况介绍，发到邮箱。回复的时候我说明，2k太低。还未收到回复~

总结：

基本的礼貌呢？收到邮件不应该回复吗？企业了不起？

3、北京国信清软科技有限公司宁波分公司（宁波）

时间：2017年1月3日 现场面试

进门……”两个人面试还是一个人？”，顿时感觉被坑了。填单子，没有技术问题，low 的一比，房间一点点大，像个前台不是公司。了解后是这样子的，做网游说我没经验，要先培训，去杭州，培训有钱先给你钱 3k / 月，接下来还钱，都特么是套路。

总结

培训机构很累

4、浙江企朋（杭州）

时间：2017年1月6日 视频面试

是面试前的晚上来的电话，我不知道那个就是人家的 HR，听起来像是个同学，问出来的问题就像是在念稿子，感觉年纪跟我差不多。不过聊得倒是蛮开心的…..最奇怪的问题是，竟然会问星座，哈哈，不过我喜欢。
这也是我目前为止看到的最合适我的一家公司，团队年轻产品丰富，招的又是物联网方向的实习生，看了他们的网站透露出一种年轻的气息，公司文化也不错，看拉勾上面的面试评价说办公环境非常的好。

在面试前，做了一些关于他们的公司的笔记，产品啊方向啊。但是一想到是技术面试，心还是有点慌慌的，自己清楚自己的底子薄弱。
面试时间挺长的，大概持续了四十分钟。到最后，面试的前辈去签字什么的。开始问的是JavaEE方向的，后来问了Netty、线程这块，问了很多很多问题，整套下来真的是精疲力尽。

4.1、网站、框架

最喜欢的网站是哪几个？淘宝、12306，在处理高并发的同时，又完美的处理订单，不容易。
是否看过介绍淘宝架构的资料？看过，但是水平有限没看懂，一方面现在看这个还早。

下面就是问框架：

DButil是什么？ Aapche 组织的开源 Dao 层轻量级小框架
另外的 Dao 层框架还用过哪些？
Hibeinate 会用么？什么时候用到这个的？
Spring 用过么？ 简单的 IOC、DI 会，注解还没深入的了解过
Struts2 用过么？ 模仿 Struts2 写过小框架，用反射机制+前后端约定

4.2、Netty、Redis、线程、TCP

还有问的是项目，首先问了消息服务器。说我负责什么？团队是怎么分工的？ 后来不知怎么的就问道了 Netty（估计是我自己挖的坑）。问我 为什么选择 Netty？我特么竟然说了网上的人都说 Netty 好，所以选择 Netty。

有没有用过其他的框架？答了 Mina。
有没有比较过两者之间的性能差异？自己有没有进行过一些测试？
项目是进行测试的？我答：测试需要盒子，还在跟盒子那边商量。
怎么保证你们写的代码能上线和安全性？ 哎……这里就被….
有没有自己动手尝试写类似 Netty 的框架？当然没有
自己写的话，可能会用到哪些类？这里回答的是最失望的犹豫了一下，答了 Channel、ByteBuf、Selector。

接着就开始问线程：

线程的状态有哪些？ 大致讲了，产生、运行、阻塞、睡眠
状态之间是怎么相互转换的？ 简直了….
锁的作用以及存在意义？ 多线程同时修改线程间共享变量时会产生数据不一致错误，加锁可以使动作串行化
进程间通信？ 我特么竟然说，不知道！数据库、Socket、文件都是啊！
线程间通信？ 使用全局变量，肯定错误…

后来是开始问到 Redis：

为什么选择 Redis? MySQL 无法承受高并发
Redis 的两种持久化方案？ 不知道…..
每天的数据量统计？ 7、8个G
如果是独立 Redis 服务器，对服务器硬件要求？ 卧槽！内存要大！其他没有什么要求

关于 TCP ：

对 TCP 有什么了解？ 三次握手，四次挥手
接入和断开的细节是怎么样的？

4.3、版本控制、项目管理、书籍

零零散散的还问了，是否看书？我说文学书、技术书都看。最近看的书是哪本？我不知道他问的是技术书….竟然答了《无声告白》，他也笑了。《深入理解 Java 虚拟机》。看到哪里？Java 内存分布那块。还说了《Java 并发编程》。

jar 包怎么管理？ 没有使用 Maven，因为项目小，Jar 手工管理
版本怎么控制的？一直在主干上开发吗？ 是的
分支创建过没有？分支存在的意义，你觉得哪些情况下需要创建分支？ 按模块分任务的时候，各自完成自己的，主干由核心人员维护。对吗？这都是我瞎扯的。
Git 用过吗？给你带来的最大好处是什么？ 云同步，即使是其他小伙伴在各自家里开发，也能很好的管理项目版本
Git、SVN对比的优缺点，你怎么看？

4.4、个人发展、结尾

自己以后的方向？大数据、物联网
对公司有什么期望？
对同事有什么期望？
还有什么问题要问？

总结

被刷
